# CONTRIBUTING — Руководство по добавлению когов

## Для фракций

1. Сделайте форк репозитория.
2. Создайте директорию для вашей фракции (если её ещё нет) или используйте существующую.
3. Разместите код кога в соответствующей поддиректории:
   - `cogs/` — слэш-команды
   - `events/` — обработчики событий
   - `tasks/` — фоновые задачи
   - `dataUtils/` — файлы для работы с базой данных (через Prisma).
   - `utils/` — вспомогательные файлы, которые вы можете сделать для вашего бота.
 4. Добавьте/обновите `config.js` с необходимыми настройками.
 5. Откройте Pull Request в основной репозиторий.

## Префиксные команды

Если когу нужна поддержка префиксных команд (через `PrefixHandler` бота), экспортируйте объект `prefix` рядом со слэш-командой:

```js
module.exports = {
  data: new SlashCommandBuilder()
    .setName('команда')
    .setDescription('Описание команды'),

  async execute(interaction) {
    // Ваш обработчик
  },

  prefix: {
    command: 'команда',          // Основная команда
    aliases: ['alias1', 'alias2'], // Псевдонимы вашей команды, например: command
    async run(message, args, client) {
      // message — обычное сообщение
      // args — массив аргументов после команды
      // client — экземпляр бота
    }
  }
};
```

Подробнее см. `cogs/hello.js` в вашем боте — пример с `/привет` и префиксной `привет`.

## Доступные пакеты

В боте предустановлены следующие пакеты (полный список — `package.json` в корне репозитория):

| Пакет | Назначение |
|-------|-----------|
| `discord.js` | Работа с Discord API (слэш-команды, события, компоненты) |
| `@prisma/client` / `prisma` | ORM для работы с базой данных |
| `winston` | Логирование |
| `luxon` / `moment-timezone` | Работа с датой и временем |
| `axios` / `node-fetch` | HTTP-запросы |
| `dotenv` | Загрузка переменных из `.env` |
| `uuid` | Генерация UUID |
| `cors` / `express` / `body-parser` | HTTP-сервер |

Если для кога требуется пакет, которого нет в списке — откройте **тикет** для обсуждения перед добавлением.

## Требования к коду

- Код пишется на **JavaScript (Node.js)** с использованием **discord.js**.
- Соблюдайте структуру директорий, указанную выше.
- Используйте только те пакеты, что уже есть в боте. Добавление новых — через тикет.
- Для локального тестирования с БД можете доустановить пакеты через `npm install <пакет>`, но **не редактируйте** `package.json` и `package-lock.json` — изменения в них вносятся только через ревью.
- Не используйте `eval()`, `Function()`, динамический импорт произвольных путей и другие небезопасные конструкции.
- Не загружайте файлы, не относящиеся к когу (`.env`, токены, логи и т.д.).
- Фоновые задачи не должны чрезмерно нагружать бота (интервал и частота должны быть адекватны задаче).

## Работа с базой данных

- Доступ к БД — через Prisma-клиент (`@prisma/client`), подключение берётся из `process.env`.
- Импортируйте Prisma-клиент стандартным способом; указывать свои строки подключения в коде кога **запрещено**.
- Файлы с запросами к базе размещайте в `dataUtils/` вашей фракции.

Пример файла в `dataUtils/`:

```js
const { prisma } = require('../prisma/client');

async function getUsersByFaction(factionId) {
  return prisma.user.findMany({
    where: { factionId }
  });
}

async function createUser(data) {
  return prisma.user.create({ data });
}

module.exports = { getUsersByFaction, createUser };
```

> **Важно:** если вашему когу требуется новая таблица или изменение схемы — после тестирования поместите файл `schema.txt` с необходимыми изменениями Prisma-схемы в **корень бота** (папка `фракция/`). Отдельно укажите это в PR.

## Процесс ревью

1. Модераторы проверяют код на наличие вредоносных или неоптимальных конструкций.
2. Если код чист — Pull Request принимается и мержится.
3. Если есть замечания — модератор оставляет комментарии в PR.
4. При обнаружении грубых нарушений безопасности PR отклоняется без возможности повторной отправки той же версии.

## Запрещено

- Любые действия от имени бота без явного разрешения (рассылка, спам, удаление сообщений).
- Сбор и утечка пользовательских данных (ID, токены, содержимое ЛС).
- Использование `process.env` без предварительного обсуждения через тикет.
- Изменение конфигурации самого бота (через `client` и т.д.).
- Обфускация или минификация кода.
